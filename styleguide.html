The danicfilip.com Style Guide

Yes, even a minimalist website can have a style guide and everyone should have one. It allows one to build a cohesive experience right from the start. As the site grows, so does the style guide. It is, thus, important to have a reference.

Colors

The site does not have too many colors, but there are a few worth noting.

  • Gray26 (#424242), used for the default text color
  • White Smoke (#f6f5f5), used for the background
  • Flat Silver (#bdc3c7), used for borders and shadows that contrast the background.
  • Summer sky (#30a9de), used for accents, buttons etc.
  • Dodger blue (#118df0), used for accents, buttons etc.

Typography

Verdana is the font family of choice for all of the typography on the site (apart from code blocks which are monospaced). Sixteen (16) pixels is the default font size, applied to the body directly.

Links

Links do not differ too much from regular text. They inherit the default Browser styles and behavior. There is also the class .link-button which styles a link as a button element.

Code Snippets

Inline code snippets that are wrapped in the <code> tag have a unique color and border around them. Full snippets will likely be embeded as Github gists or CodePen pens.

Lists

Unordered lists with a simple <ul> tag:

  • Coffee
  • Tea
  • Milk

Ordered lists with a simple <ol> tag:

  1. Coffee
  2. Tea
  3. Milk

Nested lists will inherit the default browser behavior or user preferenes.

Images

Images inside of the articles and content are wrapped inside a <figure> element. The .image-figure--default" will limit the maximum width to 600px. A caption is usually included as well.

The .image-figure--small" will limit the maximum width to 400px.

Helpers

Helper classes are a series of composable classes that can be applied to any element. Currently, this is only the .content-center class which align the text to the center position.
